# SoulDoc V5 收尾补全说明

> 2026-04-16 收口补充：
> `Space` 是内容主容器；
> `Collection` 是 Scope 下的可选分组；
> `Site / Section` 属于发布层；
> 产品层统一写作“GitHub 同步”。

## 1. 目的

本文用于对照 `SoulBook/3-Soulbook-2.0` 的 12 份设计文档，确认 `SoulDocV5` 原型是否遗漏核心功能、架构和逻辑，并记录本轮收尾补全结果。

原则：

- 不改变既有页面样式
- 不改变整体布局结构
- 不新增一套新的交互范式
- 只在 `V5` 现有页面体系内补强表达与逻辑完整性

---

## 2. 本轮重点补齐内容

### 2.1 从“文档系统”补齐为“AI 原生知识工作台”

已补入：

- `对话转文档` 工作流入口
- `草稿 -> 版本 -> CR -> 发布` 的治理链表达
- AI 与人类平权协作下的执行边界说明

对应页面：

- `index.html`
- `docs.html`
- `editor.html`
- `ai-tasks.html`
- `ai-tools.html`

### 2.2 补齐 Account 根节点与空间类型模型

`3-Soulbook-2.0` 明确强调 `Account -> Space -> Document` 的上层边界，以及 `Personal Space / Team Space` 的差异。V5 本轮进一步收口为：`Collection` 是 Scope 下的可选分组，`Site / Section` 是独立的发布层。

已补入：

- `Account` 作为账户与订阅根节点
- `Personal Space / Team Space` 的职责差异
- `公共 / 私有` 作为可见性模型
- 空间类型与治理链、审批链、审计链的关系

对应页面：

- `workspace.html`
- `spaces.html`
- `space.html`
- `profile.html`

### 2.3 补齐安全、审计、恢复、可观测性

`3-Soulbook-2.0` 的安全和后端文档里，对 `MFA / SSO / 审计 / 灾备 / OpenTelemetry / AI 安全` 讲得很完整。V5 本轮把这些能力补成了产品原型可见的系统信号。

已补入：

- 统一审计链
- 软删除 + 回收站恢复窗口
- 监控与可观测性
- AI 安全边界
- 备份 / 恢复 / 灾备的系统位

对应页面：

- `index.html`
- `space.html`
- `editor.html`
- `settings.html`
- `developer.html`
- `profile.html`

### 2.4 补齐多语言治理模型

`3-Soulbook-2.0` 的多语言设计不只是界面翻译，而是内容级多语言治理。本轮在 V5 里进一步明确了：

- `doc_group_id` 关联多语言版本
- 默认语言与回退语言
- 跨语言召回搜索
- 翻译工作流
- 翻译记忆与术语库

对应页面：

- `search.html`
- `language.html`
- `docs.html`
- `editor.html`
- `index.html`

### 2.5 补齐 AI 工具体系的 V2 优点

V2 中最重要的一点是：AI 工具不是零散按钮，而是一套可以治理、可审计、可串联的能力模型。V5 本轮继续补强了这层表达。

已补入：

- 6 大能力族
- V2 的 16 个核心工具到 V5 能力族的映射
- `dialogue_to_document / analyze / optimize_for_seo / version` 等典型能力的落点
- 工具执行结果进入 `AI 任务中心 / 通知 / 版本 / CR`

对应页面：

- `ai-tools.html`
- `ai-tasks.html`
- `developer.html`

---

## 3. 当前 V5 已覆盖的核心架构

截至本轮，`SoulDocV5` 原型已覆盖以下主链：

1. `Account / Workspace / Space / Document Group / Document(locale)`
2. `Human User / AI User` 平权用户模型
3. `Role / Permission / Execution Boundary`
4. `Draft / Version / Change Request / Publish`
5. `Public / Private Space`
6. `Language Variant / Fallback / Cross-lingual Search`
7. `AI Task / AI Tool / Workflow Engine / Event Bus / Observability`
8. `SEO / Public Site / GitHub 同步 / Developer Platform`

---

## 4. 仍然没有展开到原型细节的部分

以下内容在 `3-Soulbook-2.0` 文档里有深入设计，但当前 V5 仍停留在“系统位表达”，没有继续细化到独立页面或复杂交互：

- 完整订阅计费体系
- 全量风控与威胁检测面板
- 更细的模型完整性校验流程
- 大规模企业级部署、容灾与多区域基础设施细节
- 更完整的多行业场景模板库

这些不是遗漏，而是原型阶段主动收敛，避免页面膨胀。

---

## 5. 本轮收尾结论

结论是：`SoulDocV5` 已经具备从 `3-Soulbook-2.0` 过渡到真实产品设计稿的基础完整性。

它现在不是单纯的知识库 UI，而是一套具备以下特征的 AI 原生知识平台原型：

- AI 与人类平权协作
- 从对话到文档的生产闭环
- 从编辑到发布的治理闭环
- 从多语言到公开站点的分发闭环
- 从权限到审计的安全闭环

如果下一阶段进入前后端研发，建议直接以本目录 `01-07` 文档为第一版产品与技术基线。
