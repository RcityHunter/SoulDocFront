# SoulDocV5 接口清单

> 2026-04-16 收口补充：
> `Collection` 是 `Scope` 下的可选分组；创建 `Space` 时 `collection_id` 可以为空；
> 产品文档统一使用“GitHub 同步”，技术实体和数据表仍可保留 `git_sync` 命名。

## 1. 通用约定

- Base URL：`/api/v1`
- 认证：`Authorization: Bearer <token>`
- 数据格式：`application/json`
- 长任务：统一返回 `task_id`
- 高风险写操作：统一遵循 `Preview -> Confirm -> Commit -> Audit`
- 公开阅读页接口：`/public/*`
- 能力清单接口：`/.well-known/*`

说明：

- `AI` 不是单独的系统角色，而是 `user_type=ai` 的用户
- 因此接口设计优先围绕 `user`、`role`、`space_member` 展开，而不是额外引入 `agent` 体系
- 个人文档与组织文档分属不同载体：`personal_workspace` 与 `organization`
- 产品文案统一使用 `发布站点` 对应技术实体 `site`
- 产品文案统一使用 `内容集合` 对应技术实体 `collection`
- `站点分区` 对应 `site_section`

## 4. Personal Workspace、Organization、内容集合（Collection）、Space、发布站点（Site）与站点分区（Section）

### 4.1 Personal Workspace / Organization

- `GET /me/workspaces`
- `GET /personal-workspace`
- `PUT /personal-workspace/preferences`
- `GET /organizations`
- `POST /organizations`
- `GET /organizations/:orgId`
- `PUT /organizations/:orgId`
- `GET /organizations/:orgId/members`
- `POST /organizations/:orgId/members/invite`
- `PUT /organizations/:orgId/members/:memberId`
- `DELETE /organizations/:orgId/members/:memberId`

说明：

- 个人工作区用于承载私有草稿、偏好和个人空间
- 组织用于承载组织成员、内容集合、组织空间、模板库和发布站点

### 4.2 发布站点（Site）

- `GET /sites`
- `POST /sites`
- `GET /sites/:siteId`
- `PUT /sites/:siteId`
- `DELETE /sites/:siteId`
- `GET /sites/:siteId/navigation`
- `PUT /sites/:siteId/navigation`
- `GET /sites/:siteId/sections`
- `POST /sites/:siteId/sections`
- `PUT /sites/:siteId/sections/:sectionId`
- `DELETE /sites/:siteId/sections/:sectionId`
- `GET /sites/:siteId/spaces`
- `POST /sites/:siteId/spaces/bind`
- `DELETE /sites/:siteId/spaces/:bindingId`

说明：

- `site` 需要返回 `organization_id`
- 创建发布站点时必须指定归属组织

### 4.3 内容集合（Collection）

- `GET /collections`
- `POST /collections`
- `GET /collections/:collectionId`
- `PUT /collections/:collectionId`
- `DELETE /collections/:collectionId`
- `POST /collections/:collectionId/spaces/:spaceId/bind`
- `DELETE /collections/:collectionId/spaces/:spaceId/unbind`

说明补充：
- `space` 可以不进入任何 `collection`，以顶层 Space 形式直接挂在 `scope` 下

说明：

- `collection` 属于 `personal_workspace` 或 `organization`
- `site` 不直接拥有 `collection`

### 4.4 Space

- `GET /spaces`
- `POST /spaces`
- `GET /spaces/:spaceId`
- `PUT /spaces/:spaceId`
- `DELETE /spaces/:spaceId`
- `GET /spaces/:spaceId/overview`
- `GET /spaces/:spaceId/settings`
- `PUT /spaces/:spaceId/settings`

说明补充：
- `POST /spaces` 支持可选 `collection_id`；为空表示创建顶层 Space

说明：

- `space.visibility` 仅允许 `public` / `private`
- `space.owner_scope_type` 仅允许 `personal_workspace` / `organization`
- `space.owner_scope_id` 用于标识个人工作区或组织归属

## 12. 前端优先接入顺序

### P0

- 安装、登录、当前用户
- Scope / Space / Collection（可选） / Site
- 文档树、文档详情、文档编辑
- 版本列表、变更请求
- 搜索、AI 任务列表
- 语言版本读取
- SEO、发布目标、公开阅读页读取

### P1

- 文档块、标签、文件、评论
- 成员、角色、自定义权限
- GitHub 同步、Webhook、能力清单

### P2

- 更细粒度的发布分析
- 站点级运营数据
- 外部连接器扩展
