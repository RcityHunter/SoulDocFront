📐 模板中心

模板是 Space 内的标准起稿器，不改变治理链。它解决“如何快速起草、如何统一结构、如何批量换版”的问题。

1. 选择模板

按归属、Space、语言、文档类型筛选 PRD、API、帮助中心、发布说明等模板。

2. 替换变量

模板变量可绑定 Space、语言、负责人、发布日期、产品版本与 SEO 元数据。

3. 生成草稿

生成结果仍然进入普通文档链，继续走版本、审批、语言版本与发布。

4. 升级模板

模板本身也有版本，支持预览影响面、批量迁移和 GitHub 回灌。

模板来源与归属

个人模板
用于个人工作区快速起稿，适合私人写作结构、调研笔记、会议纪要等。
组织模板
由组织统一维护，适合 PRD、API、FAQ、帮助中心、运维手册等标准化内容。
Space 默认模板
每个 Space 可以指定默认模板集，新建文档时直接按当前 Space 的模板习惯起稿。
发布模板
模板还能绑定发布站点栏目、SEO 片段、语言策略和 GitHub 同步规则。

模板制作方式

把一篇成熟文档转成模板，保留目录树、说明块和变量占位符。
由 AI 基于最佳实践自动生成模板草稿，再由人类或高权限 AI 审核。
从 GitHub 中已有 Markdown 目录导入模板，作为 docs-as-code 团队的标准基线。
模板升级前先给出影响预览，明确哪些文档可批量升级、哪些文档保持旧版。

模板更换与升级

模板升级不直接改写现有文档，而是先生成升级预览，列出新增章节、弃用字段和受影响语言版本。
Space 可以切换默认模板集，但只影响新建文档；已有文档通过“应用模板升级”进入版本与审批链。
组织模板支持灰度：先对单个 Space 生效，再放大到整个组织。

与 AI / GitHub 的关系

AI 用户可以根据内容目标生成模板草稿，但模板发布仍要经过人类或高权限 AI 审核。
模板目录可以同步到 GitHub 仓库，适合 docs-as-code 团队统一模板版本与代码评审。
模板变量、提示词、样例片段和 SEO 片段都能作为独立对象进入版本治理。
