产品需求文档 PRD v2.0

SoulDoc 的多语言采用行业常见的“同一文档组 + 每语言独立版本”模式：每个 locale 各自保存、各自发布，用同一个文档组 ID 关联，并支持默认语言回退与并行校对。

zh-CN 主语言 en-US 已发布 ja-JP 待复审 es-ES 未创建
行业通用做法
1. 每种语言是一份独立文档，可独立保存、独立发布。
2. 各语言通过统一的 `doc_group_id` 关联，而不是把所有文本塞进一份长文档。
3. 默认语言作为源版本，其它语言记录翻译状态、更新时间、是否落后于源文。
4. 阅读端提供语言切换；编辑端提供并排对照与缺失回退。

Product Requirements Document PRD v2.0

SoulDoc is an AI-native documentation system for knowledge creation, organization, collaboration, and multilingual delivery. Each locale is versioned independently while remaining linked inside the same document group.

更新者：Aurora AI · 昨天 17:20

翻译工作流

1. Detect
源语言变更后，自动识别受影响 locale 与变更块。
2. Draft
由 AI 先生成翻译草稿，但不直接覆盖已发布版本。
3. Review
进入待复审队列，由人类或具备权限的 AI 用户完成校对与发布。

翻译记忆与术语库

相同段落优先命中翻译记忆缓存，避免同一篇文档反复重译。
产品名、角色名、权限名与品牌措辞进入术语库，保证不同语言版本的一致性。
跨语言搜索会共享语义索引，但阅读端仍以当前 locale 或回退语言输出正文。
