仓库配置

GitHub / souldoc-knowledge
默认分支 `main`，预览分支 `preview/*`，同步文档、模板和提示词片段。
同步策略
Markdown 主内容双向同步；模板、SEO 配置、公开访问策略和站点结构优先以 SoulDoc 发布链为准。
权限边界
AI 用户可创建预览草稿和建议补丁，但不能直接推送发布分支。

冲突处理

保留仓库版本、SoulDoc 草稿版本和 Diff 预览三份结果，不把冲突直接吞掉。
技术团队可以在仓库侧解决 Markdown 结构冲突，内容团队可以在可视化侧处理元数据、SEO 和语言版本差异。
通知中心与 Webhook 会同步收到 `git_sync.conflict` 回执。
